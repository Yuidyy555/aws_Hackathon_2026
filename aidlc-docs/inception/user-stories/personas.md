# ペルソナ定義

## Persona 1: 主ペルソナ（エンジニア職）
- **名前（仮）**: 佐藤 直人
- **役割**: ソフトウェアエンジニア（個人 contributor）
- **状況**: リモート中心で、Slackやレビューコメントでの対人コミュニケーションが多い
- **課題**:
  - 伝えたい技術的意図はあるが、表現トーンで誤解や摩擦が起きる
  - 感情を抑えてビジネス文に整える作業が心理的コストになる
  - 返信文作成に時間がかかり、開発集中が削られる
- **期待価値**:
  - 感情と意図を保ったまま、自然なビジネス文に変換される
  - Slackでそのまま使える返信案を短時間で得られる
  - 履歴から過去の言い回しを再利用できる

## Persona 2: 副ペルソナ（評価者/レビュー観点）
- **名前（仮）**: 山本 由紀
- **役割**: チームリード/評価者
- **状況**: 複数メンバーのアウトプット品質と協働の円滑性を見ている
- **課題**:
  - 本人の思考力は高いのに、文面トーンで評価がぶれやすい
  - フィードバックに対する応答品質にムラがある
- **期待価値**:
  - 返信文が丁寧かつ意図明瞭で、対話品質が安定する
  - メンバーが過度に消耗せず、意思疎通が継続できる

## Persona 3: 追加ターゲット（コミュニケーションに悩む当事者）
- **名前（仮）**: 中村 彩
- **役割**: プロジェクト推進担当（非エンジニア職）
- **状況**: 複数部署との調整や依頼返信が多く、Slackでの文面作成に強い負荷を感じている
- **課題**:
  - 感情が先に立つ場面で、角が立たない文面に整えるのが難しい
  - 相手に配慮しすぎて意図が弱くなり、依頼や相談が伝わりにくくなる
  - 自己評価や1on1向けの言語化に時間がかかり、心理的負担が高い
- **期待価値**:
  - 意図を失わずに、受け手に伝わる自然なビジネス文へ変換できる
  - 返信作成の迷いを減らし、やり取りのストレスを下げられる
  - 履歴を使って、自分に合う言い回しを再利用できる

## ペルソナと価値仮説
- 主価値は「自分の感情・意図を汲み取った自然なビジネス文章」の生成
- MVPの最重要KPIは上記主価値の体験達成度
- 速度・継続利用意向・共感性は副次KPIとして追跡する
